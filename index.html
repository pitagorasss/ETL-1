RUNNING
01 · Extração

Leitura automatizada de múltiplas fontes com Pandas e controle de memória.

  • vendas.csv via pd.read_csv()
  • gerentes.xlsx via pd.read_excel()
  • low_memory configurável
  • Retorno como DataFrames
02 · Transformação

Limpeza, padronização e tratamento dos dados brutos.

  • Drop coluna DATA_BASE
  • fillna('Online') em LOJA
  • dropna() nas demais colunas
  • Datas → ISO 8601
  • Dedup por ID (keep='first')
  • str.strip() + str.title()
03 · Carga

Load em SQLite. Banco relacional nativo do Python

  • Tabelas criadas automaticamente
  • FOREIGN KEY: vendas → gerentes
  • Inserção via df.to_sql()
  • Arquivo vendas.db

Problemas e Soluções

🕳️
Nulos

Valores Nulos

Colunas como LOJA com registros vazios comprometendo análises regionais.

✓fillna('Online') + dropna() nas demais colunas.
📅
Datas

Datas Inconsistentes

Formatação heterogênea e horários desnecessários dificultando agrupamentos temporais.

✓Conversão e padronização para datetime, extração ISO 8601.
♊
Duplicatas

Registros Duplicados

IDs duplicados distorcendo contagens e métricas de receita por produto.

✓drop_duplicates(subset=['ID'], keep='first').
🔤
Strings

Nomes Não Padronizados

Espaços extras e capitalização inconsistente quebrando joins e agrupamentos.

✓str.strip() + str.title() em todos os campos.
🔗
Integridade

Sem Integridade Relacional

Vendas e gerentes sem vínculo formal, impossibilitando análises conjuntas.

✓FOREIGN KEY (id_gerente) no esquema SQLite.
🗂️
Formatos

Formatos Heterogêneos

CSV e Excel sem processo unificado de ingestão, dificultando manutenção.

✓Pipeline único via ETL.py, abstraindo as fontes.
🧮
Tipos

Tipos de Dados Incorretos

Colunas numéricas importadas como string impedindo operações aritméticas e agregações corretas.

✓Conversão explícita de dtypes via Pandas após a extração.
🏗️
Esquema

Ausência de Esquema Formal

Dados sem estrutura relacional definida, sem constraints de tipo ou chaves primárias garantidas.

✓Esquema declarado com PRIMARY KEY e FOREIGN KEY no SQLite.

CÓDIGO EXECUTOR 
import pandas as pd
import numpy as np
from comp import extracao, transformacao
from database import criar_tabelas, carga

def etl():
    # ── 1. Extração ─────────────────
    df_vendas, df_gerentes = extracao(
        low_memory=False
    )
    # ── 2. Transformação ────────────
    df_vendas, df_gerentes = transformacao(
        df_vendas, df_gerentes
    )
    # ── 3. Carga ────────────────────
    criar_tabelas()
    carga(df_vendas, df_gerentes)
    return df_vendas, df_gerentes

if __name__ == '__main__':
    df_vendas, df_gerentes = etl()

def transformacao(df_vendas, df_gerentes):
    # Remove coluna desnecessária
    df_vendas = df_vendas.drop(
        columns=['DATA_BASE']
    )
    # Trata nulos em LOJA
    df_vendas['LOJA'] = (
        df_vendas['LOJA'].fillna('Online')
    )
    df_vendas = df_vendas.dropna()
    # Formata datas → ISO 8601
    df_vendas['DATA'] = (
        pd.to_datetime(
            df_vendas['DATA'],
            format="%Y-%m-%d"
        ).dt.date
    )
    df_vendas = df_vendas.drop_duplicates(
        subset=['ID'], keep='first'
    )
    return df_vendas, df_gerentes

def criar_tabelas():
    conn = criar_conexao()
    cursor = conn.cursor()
    cursor.execute('''
        CREATE TABLE IF NOT EXISTS vendas(
            id_venda   INTEGER PRIMARY KEY,
            data_venda VARCHAR(10),
            produto    VARCHAR(20),
            quantidade INTEGER,
            preco_unitario REAL,
            valor_total    REAL,
            id_gerente INTEGER,
            FOREIGN KEY (id_gerente)
                REFERENCES gerentes(id_gerente)
        )
    ''')
    conn.commit(); conn.close()

def carga(df_vendas, df_gerentes):
    conn = criar_conexao()
    df_gerentes.to_sql('gerentes', conn,
        if_exists='replace', index=False)
    df_vendas.to_sql('vendas', conn,
        if_exists='replace', index=False)
    conn.close()
